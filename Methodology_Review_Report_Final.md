## 一、系統設計總覽與評估基準確認

### 1.1 研究設計摘要

| 項目 | 內容 |
|------|------|
| 數據範圍 | 20年日資料 |
| 異常定義 | T日起22個交易日內，USDTWD最大升值幅度超過 3% |
| 訓練期 | 前15年 |
| 驗證期（OOS） | 後5年 |
| 閾值來源 | 回測優化 |
| 觀察窗口 | T-22 至 T+11（指標採集僅使用 T 日及以前） |

### 1.2 系統的兩層架構

經過討論，你的系統實質上是一個**兩層預警架構**，這是一個設計合理的結構：

```
第一層：聯集（至少一個指標觸發）
  → 目標：早期偵測，覆蓋廣度優先
  → 表現：20年18次大升值事件中，捕捉到 15次（Recall = 83%）
  → 用途：提示風管人員開始關注，進入觀察模式

第二層：交集（三個指標同時觸發）
  → 目標：鎖定 5%+ 的重大升值，精確度優先
  → 設計原則：寧可漏報，不要頻繁誤報
  → 當前精確率（Precision）：約 60%
  → 用途：觸發主動風控行動
```

### 1.3 關於 18 次事件的背景詮釋

20年出現 18 次符合定義的重大升值事件，看似樣本稀少，但必須放在台灣的制度背景下理解：

> 台灣央行（CBC）是全球干預最積極的央行之一，長期壓制台幣的單向升值幅度。因此，USDTWD 出現 3%+ 的窗口內最大升值，意味著升值壓力已大到連 CBC 都無法完全壓制。**這18次事件不是「台幣升值了18次」，而是「市場升值壓力突破了CBC干預防線18次」。** 在這個詮釋下，18次反映的是真正的極端事件，樣本稀少是正確且預期的。

## 二、方法論優點確認

以下是這套研究設計中值得肯定的部分：

| 優點 | 說明 |
|------|------|
| **選用NDF衍生品市場** | NDF不受CBC在岸管制，反映的是全球市場對台幣的真實預期定價，是最接近「不被干預的市場信號」的來源 |
| **1M-12M 年化成本利差** | 短端NDF成本高於長端，代表市場預期壓力集中在近期，這個設計捕捉了「短期爆發性升值預期」，而非長期趨勢 |
| **年化成本的標準化** | 用年化成本而非絕對點數，使不同時期的數據具備跨期可比性，統計上正確 |
| **CVIX 採動態閾值** | 用移動平均+2倍標準差而非固定值，能自動適應市場波動率的結構性變化，優於固定閾值 |
| **AND邏輯的設計意圖明確** | 交集邏輯不是用來覆蓋所有事件，而是過濾出更大幅度的波動——這是清晰的設計選擇，不是缺陷 |
| **OOS 驗證** | 15年訓練 + 5年驗證的框架，具備基本的樣本外驗證，避免純樣本內過擬合 |

## 三、現存問題與改善建議

### 問題一：交集邏輯的 40% 誤報率——需要釐清「誤報」的本質

**現況**：三個指標同時觸發後，有 40% 的情況未達到 5%+ 的重大升值。

**建議視角的調整**：

在採取任何模型改動之前，應先回答一個問題：**那 40% 的「誤報」，USDTWD 實際走了多少？**

```
可能情境A：台幣升值了 2~4%（接近但未達 5% 門檻）
  → 這不是真正的誤報，而是標籤定義造成的假象
  → 系統實際上偵測到了真實的升值壓力，只是幅度略小
  → 改善方向：考慮將第二層門檻調整為 4%，或加入分級警示

可能情境B：台幣幾乎沒動（升值 < 1%）
  → 這才是真正的誤報，代表信號偶爾在無異常時觸發
  → 改善方向：需要加入過濾條件（見下方建議）

可能情境C：兩種情況都有，比例各半
  → 系統有部分真實的誤報需要處理，但問題不如表面嚴重
```

**建議行動**：對那 40% 的觸發事件，標記當下 USDTWD 的後續最大升值幅度，區分上述三種情境，再決定改善方向。

---

### 問題二：CVIX 在「風險偏好上升型台幣升值」情境中的結構性失效

**現況**：CVIX 是三個指標中最常沒有觸發的一個。

**根本原因**：

CVIX（DB FX Volatility Indicator）衡量全球外匯市場的整體波動率，其核心驅動是**市場恐慌與風險趨避**。然而，台幣最大幅度的升值往往發生在以下情境：

```
情境類型A：Risk-on 驅動（風險偏好上升）
  外資大買台股 + 半導體景氣熱絡 → 資金持續流入台灣 → 台幣升值
  此時 CVIX 偏低（市場平靜），但台幣升值壓力最大
  → CVIX 此情境下是反效果的指標

情境類型B：危機後復甦（2020下半年）
  疫情衝擊後市場反彈 → 外資回流亞洲 → 台幣急升
  此時 CVIX 從高位快速下降，信號反轉，觸發邏輯混亂

情境類型C：CBC 防線潰決
  累積升值壓力過大，CBC 被迫放手 → 快速升值
  此時 CVIX 可能尚未反映，因為觸發點不是外部恐慌
```

**改善方向一（建議優先）：以 USDTWD 隱含波動率取代 CVIX**

使用 **USDTWD 1個月期 ATM 隱含波動率（1M IV）** 取代 CVIX：
- 不帶方向性，純粹衡量市場預期台幣的波動幅度
- 直接針對 USDTWD，不受無關的 EUR/JPY 等貨幣波動稀釋
- 當市場預期台幣大升值時，買入 USD Put（台幣 Call）的需求增加，隱含波動率上升，能捕捉 Risk-on 情境

**改善方向二（備選）：CVIX 改為反向或條件性使用**

將 CVIX 改為「排除條件」而非「觸發條件」：
```
舊邏輯：CVIX > MA + 2σ 才觸發（偵測恐慌）
新邏輯：CVIX < MA + 1σ（排除全球系統性危機情境，只保留正常升值壓力事件）
```
此設計的邏輯是：若 CVIX 極度偏高，代表是全球系統性危機，而非台幣特有升值壓力，可能需要不同的處理框架。

**改善方向三（資料允許的情況）：USDTWD 25-Delta Risk Reversal**

台幣選擇權市場的 **Risk Reversal（RR）**——USD Put 隱含波動率減去 USD Call 隱含波動率——直接反映市場對台幣升值或貶值的方向性押注。RR 若大幅轉向負值（市場買入更多台幣升值的保護），通常領先於實際升值。

### 問題三：1M NDF 年化成本的絕對閾值在不同利率環境下會失效

**現況**：閾值設定為 1M NDF 年化成本 > 8%（固定值）。

**問題**：NDF 年化成本的基準線隨美元利率環境大幅變動：

```
Fed 利率 0%（2020~2021年）：
  1M NDF 年化成本 8% → 超出正常水準極多 → 確實是強烈信號

Fed 利率 5%（2023~2024年）：
  1M NDF 年化成本 8% → 相較無風險利率僅高出 3% → 信號強度大幅稀釋
```

**建議改善：改用 NDF 超額溢價（Excess Premium）**

```
NDF 超額溢價 = 1M NDF 年化成本 - 當期美元1個月 LIBOR/SOFR

新閾值邏輯：NDF 超額溢價 > X%（X 由回測重新優化）
```

這個調整讓指標在不同利率週期下保持一致的信號強度，且概念上更精確：它衡量的是「市場願意為台幣升值保護額外支付多少溢價」，而非被利率水準污染的絕對成本。

### 問題四：競爭對手匯率的互補性尚未充分評估

**現況**：日圓、韓元、人民幣、歐元等已研究，但未納入最終指標。

**建議**：不必作為主要觸發指標，但可作為**第一層聯集信號的補充**，或作為交集觸發後的「信號強化確認」：

```
建議的使用方式：
  若交集觸發時，USDKRW 同期也出現同向升值壓力信號
  → 信號評估升級（區域性資金流動，而非台灣特有事件）

  若交集觸發時，USDKRW 無任何信號
  → 保持原信號等級，可能是台灣特有事件（如外銷訂單爆發）
```

**USDKRW 作為確認指標的理由**：韓元是亞洲外匯市場流動性最高的NDF貨幣，往往比台幣更早反映全球資金流向亞洲的趨勢。

## 四、小樣本預警系統的設計建議

這是本套系統最根本的統計限制，值得單獨討論。

### 4.1 小樣本的核心統計問題

20年18次事件，意味著：
- 平均每年不到1次
- 每個統計結論（83%、60%）的**95%信心區間極寬**
- 加入或移除2~3個事件，就可能讓命中率從83%變成72%或降至67%

這是台幣市場的結構性限制，無法透過「更好的模型」解決，只能透過方法論設計來管理。

### 4.2 小樣本環境下的最佳實務

**原則一：放棄「精確的命中率」，改用「信號強弱排序」**

在小樣本下，追求準確的命中率數字沒有意義。更有效的做法是：

> **將所有歷史事件按「信號強度」排序，確認信號越強、後續升值幅度越大的單調關係是否成立。**

若信號強度與升值幅度呈現單調正相關，即使樣本少，這個關係也是可信的——因為它不依賴「事件是否超過某個門檻」，而是依賴「排序的一致性」。

**原則二：使用無母數方法，不假設數據分布**

- 用 **Spearman Rank Correlation** 取代 Pearson 相關係數，避免對數據分布做假設
- 用 **Permutation Test（置換檢定）** 估計統計顯著性，不依賴大樣本的中央極限定理
- 用 **Bootstrap Resampling** 估計信心區間

**原則三：用「條件期望報酬」取代二元命中率**

```
舊思維：信號觸發後，有60%機率發生5%升值（二元判斷）

新思維：信號觸發後，未來22日台幣升值幅度的期望值是多少？
  → 觸發後22日平均升值：+2.1%
  → 未觸發時22日平均升值：+0.3%
  → 信號有效性 = 觸發期間相較非觸發期間的超額升值幅度（+1.8%）
```

這個框架不需要預設「門檻」，更適合小樣本分析，且對風控人員更直接有用。

**原則四：事件研究法（Event Study）**

針對18次事件，做標準的金融事件研究：

```
1. 定義事件窗口（Event Window）：T-5 到 T+22
2. 定義估計窗口（Estimation Window）：T-120 到 T-22
3. 計算「正常報酬」（用估計窗口的均值）
4. 計算「異常報酬」（事件窗口的累積超額升值幅度）
5. 用 t-test 或 Wilcoxon 符號秩檢定，測試異常報酬是否顯著異於零
```

這是學術界在小樣本事件研究的標準框架，能提供比簡單命中率更嚴謹的統計支撐。

**原則五：持續累積樣本，建立前瞻性驗證（Prospective Validation）機制**

> 過去的回測和OOS驗證終究都是「事後的」。真正的模型有效性，只有在模型**不知道未來數據**的情況下預測，再事後驗證，才能成立。

建議建立一個「預測登記制度」：
- 每次系統發出交集警示時，記錄觸發日期、信號強度
- 22個交易日後，記錄實際結果
- 每累積5次新事件，重新評估模型表現

這是醫學臨床試驗的「前瞻性登記」概念，在金融模型驗證中同樣適用。

## 六、需要明確揭露的模型限制

任何將此方法論應用於實務風控的報告，應包含以下標準揭露：

1. **小樣本限制**：模型基於20年18個極端事件，統計信度受樣本數限制，任何命中率數字均帶有寬幅的不確定性

2. **CBC 干預的不可預測性**：系統偵測的是「市場升值壓力」，而非「實際升值幅度」，兩者之間存在 CBC 干預這個不可量化的變數

3. **數據依賴性**：系統核心信號依賴 NDF 報價和 CVIX，若數據來源中斷或產品下架，系統需要重新驗證替代指標

4. **制度性結構變化**：NDF 市場結構、CBC 干預風格、全球利率環境均在20年間發生顯著變化，模型對未來結構性轉變的適應能力尚未驗證

5. **前視偏差聲明**：指標閾值及指標選擇均來自歷史回測，存在一定程度的樣本內優化，OOS 驗證期（5年）的結論應比訓練期結論給予更高的權重
