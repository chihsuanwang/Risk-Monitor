## 1. 為什麼台積電是外匯風險的核心變數

在進入台積電股價分析之前，必須先說明它與 USDTWD 風險管理的關係——這不是兩個獨立的問題，而是同一條資金流動鏈的兩個節點。

### 1.1 台積電的市場地位

| 數字 | 意義 |
|------|------|
| 台積電佔 TAIEX 權重約 30~35% | TAIEX 幾乎等於台積電指數 |
| 外資持有台積電約 70%+ | 台積電波動直接驅動外資換匯行為 |
| 台積電 ADR（TSM）全球流動性極高 | 美股盤後的 TSM 走勢是台灣開盤的領先信號 |
| 台積電市值是台灣 GDP 的 1.5 倍以上 | 單一公司對台灣資本市場的影響無出其右 |

### 1.2 台積電 → USDTWD 的傳導鏈

```
台積電股價大跌（-4%+）
        ↓
TAIEX 同步重挫（外資主導）
        ↓
外資賣出台積電 → 賣台幣、買美元（換匯回本國）
        ↓
USDTWD 短期升壓（台幣貶值壓力）
        ↓
若跌幅持續 → 外資持續撤退 → 台幣中期趨弱

反向情境（台積電大漲）→ 外資買超 → 台幣升值壓力
```

**結論：台積電股價監控不只是股票問題，它是 USDTWD 預警系統的上游感測器。**

## 2. 台積電股價下行的驅動機制

在規劃任何監控系統之前，必須先理解「誰」讓台積電跌 4% 以上。

### 2.1 觸發類型分類

我將台積電的大跌事件分為四種類型，監控策略完全不同：

#### 類型 A：半導體景氣週期下行（可預警，領先時間 1~3 個月）

```
全球半導體庫存累積
    ↓
終端需求（手機、PC、伺服器）下滑
    ↓
客戶削減訂單（蘋果、NVIDIA 下修展望）
    ↓
台積電稼動率下降 → 法說會下調指引
    ↓
台積電股價大幅下修
```

**特徵**：這類下跌通常**有跡可循**，領先指標包括記憶體報價、客戶庫存水位、SOX 指數走勢。是本監控系統最有機會提前預警的類型。

#### 類型 B：重大客戶事件（可部分預警，領先時間 1~2 週）

```
NVIDIA / 蘋果 / AMD 財報 → 展望不如預期
    ↓
市場重新估算台積電訂單量
    ↓
台積電股價下修
```

**特徵**：財報日期已知（可查日曆），但結果不可知。監控重點是**財報前的市場定價（期權隱含波動率上升）**，以及財報後的即時反應。

#### 類型 C：地緣政治衝擊（幾乎無法預警，即時反應型）

```
台海緊張升溫 / 美國對中國晶片禁令 / 中國反制措施
    ↓
台積電「地緣政治折價」擴大
    ↓
外資瞬間撤退 → 股價急跌
```

**特徵**：無法用量化指標預警，只能透過**新聞監控**和**地緣政治風險指數**做快速應對框架。任何量化模型對此類事件效果有限。

#### 類型 D：美股系統性下跌的拖累（部分可預警，領先 1 天）

```
美股大跌（Fed 鷹派、衰退擔憂）
    ↓
TSM ADR 在美股跌幅大
    ↓
台積電隔日開盤直接跟跌
```

**特徵**：TSM ADR 的美股收盤價是最直接的隔日預警信號，領先時間約 15~18 小時（美股收盤到台北開盤）。

### 2.2 各類型的歷史頻率（近10年估計）

| 類型 | 年均頻率 | 可預警性 | 監控優先序 |
|------|---------|---------|----------|
| A：景氣週期 | 1~2次（以波段計） | 高 | P1 |
| B：客戶事件 | 4~8次 | 中 | P2 |
| C：地緣政治 | 1~3次 | 極低 | P3（應急框架）|
| D：美股拖累 | 5~15次 | 高（次日） | P1 |

## 3. 方法論設計

### 3.1 異常定義

```
主要定義：台積電（2330.TW）單日收盤跌幅 ≥ 4%

輔助定義（建議同步追蹤）：
  (1) 5日累計跌幅 ≥ 6%（中期下行動能）
  (2) 單日跌幅 ≥ 2.5%，且伴隨成交量放大（> 20日均量 × 1.5 倍）
```

**為何選 4%？**
台積電近10年日報酬標準差約為 1.8~2.2%，4% 約為 2 個標準差事件，在統計上屬於低頻但有意義的異常。相較於 USDTWD 的 3% 22日定義，台積電單日 4% 的事件頻率更高（估計每年 10~20 次），統計樣本更充足，這是與 USDTWD 研究的重大差異。

### 3.2 觀察窗口設計

```
預警期（Leading Window）：T-5 到 T（觀察觸發前5日的指標走勢）
事件日（Event Day）：T
確認期（Confirmation Window）：T+1 到 T+3（觀察跌勢是否延續）

標籤邏輯：
  T日若收盤跌幅 ≥ 4% → 標記為「事件日」
  T-1、T-2、T-3、T-4、T-5 → 標記為「預警觀察日」
  模型目標：在 T-1 或更早發出預警
```

### 3.3 與 USDTWD 方法論的關鍵差異

| 維度 | USDTWD 研究 | TSMC 研究 |
|------|------------|----------|
| 事件頻率 | 20年18次（極稀少）| 預估20年200+次（充足）|
| 統計工具 | 無母數、事件研究 | 可用標準統計方法 |
| 預警時間 | 1個月 | 1天到1週 |
| 核心干擾因素 | CBC 干預 | 無結構性人為干預 |
| 信號來源 | 衍生品市場（NDF）| 多元（ADR、選擇權、基本面）|
| 樣本外驗證 | 困難（樣本少）| 相對容易（樣本多）|

### 4.1 第一層：隔夜信號（領先 15~18 小時，T-1 美股收盤時確認）

這是**最直接、最可靠**的短期預警層。

#### 指標 1.1：TSM ADR 收盤跌幅

| 屬性 | 內容 |
|------|------|
| **邏輯** | TSM 美股 ADR 與 2330.TW 在同一個市場估值框架下定價，ADR 跌幅直接反映海外投資者對台積電的即時估值調整 |
| **領先時間** | 台北開盤前 15~18 小時 |
| **警示閾值** | TSM 單日跌幅 > -3%（隔日台積電開盤大概率補跌）|
| **數據來源** | Yahoo Finance（`TSM`），每日美股收盤後可取得 |
| **注意事項** | ADR 折溢價（ADR 相對於台股換算後的溢價/折價）本身也是信號，溢價快速收窄代表需求降溫 |

**歷史統計建議**：計算「TSM ADR 跌幅 X%」與「隔日 2330.TW 跌幅 Y%」的條件分布，找到最優的警示閾值。

| 屬性 | 內容 |

## 5. 我如何思考這個問題

這一節說明分析框架背後的思考邏輯，也是評估任何新指標時應採用的判斷標準。

### 5.1 台積電不是普通的股票，有三個「特殊性」

**特殊性一：定價框架在美股，而非台股**

台積電的估值（本益比、市淨比）實際上是由美股投資者決定的，因為外資持股高達70%+。這意味著：
- **美股盤的價格發現（TSM ADR）先於台股盤**
- 台股的台積電開盤價，幾乎是「跟美股對齊」的機制，而非獨立的市場發現
- 因此，美股信號（TSM ADR、SOX、NVIDIA）比任何台灣本地指標都更重要

**特殊性二：台積電是一個「訂單能見度很高」的公司**

半導體代工的特性是：客戶需要提前 3~6 個月下訂單，台積電的稼動率和收入能見度相對清晰。這意味著：
- 突發性的「業績大幅低於預期」在正常時期相對少見
- 大跌通常來自「市場預期被重新校準」，而非真正的業績崩潰
- 記憶體報價、客戶庫存等先行指標，能在預期校準前給出信號

**特殊性三：AI 週期讓台積電的定價邏輯變得更複雜**

2023年後，台積電不只是半導體週期股，它還是 AI 基礎設施股。這帶來了：
- 傳統半導體景氣指標的預測力下降（舊的庫存週期邏輯部分失效）
- NVIDIA 的 AI 訂單展望對台積電的影響超過傳統 PC/手機客戶
- 估值溢價更高，也意味著預期落差時的跌幅更大

### 5.2 預警信號的邏輯優先序

當面對可能觸發 4% 大跌的情境時，我的思考順序是：

```
Step 1：「今晚美股怎麼樣？」（TSM ADR、SOX、NVIDIA）
  → 這是最重要的問題，因為這幾乎決定了明天台積電的開盤方向

Step 2：「未來幾週有什麼重大事件？」（法說會日曆、NVIDIA財報）
  → 事件前的隱含波動率定價會告訴你市場預期風險的大小

Step 3：「整體半導體景氣趨勢怎麼樣？」（SOX週線、DRAM報價）
  → 這決定了台積電是否在一個「易跌難漲」的結構性環境中

Step 4：「市場情緒和籌碼如何？」（Put/Call、外資買超）
  → 在前三步確認風險升高的情況下，這是確認信號

Step 5：「有沒有地緣政治尾部風險？」（人工判斷）
  → 這是所有框架之外的壓倒性因子
```

### 5.3 警告：這個市場有兩個「規律性陷阱」

**陷阱一：「壞消息已定價」效應**

當市場已充分擔憂某個風險（如法說會展望）時，往往是反向做多的信號，而非繼續做空。如果 Put/Call 比率已非常高、IV 極度擴張，代表悲觀情緒已被定價，真正的大跌通常發生在**市場還沒充分定價**時。

**陷阱二：AI 週期的非線性定價**

傳統的半導體景氣週期是可預期的庫存循環（通常12~18個月一個週期）。但 AI 算力需求的擴張是「指數型增長」，不遵循傳統週期邏輯。在 AI 週期中，傳統週期性指標（如記憶體報價下跌→景氣轉差）可能發出假信號。

**建議**：在模型中區分「傳統半導體週期模式」與「AI 週期模式」，在不同模式下使用不同的指標權重。

## 6. 監控系統規劃

### 6.1 監控頻率設計

台積電的監控必須比 USDTWD 更頻繁，因為驅動因素的時效性更短：

| 監控層級 | 頻率 | 觸發時機 | 主要信號 |
|---------|------|---------|---------|
| **隔夜快訊** | 每個美股交易日收盤後 | 自動抓取 | TSM ADR、SOX、NVIDIA |
| **每日更新** | 台灣收盤後 | 自動 | 外資買超、台積電成交量、Put/Call |
| **每週彙整** | 每週一 | 手動+自動 | 週趨勢報告、DRAM報價上傳 |
| **事件觸發** | 法說會前後 | 自動偵測 | IV 水準、ADR 即時反應 |

### 6.2 警示等級設計

```
🟢 正常模式（Green）
  → 所有指標無異常
  → 每日靜默更新，無需主動關注

🟡 觀察模式（Yellow）— 第一層或第二層任一觸發
  → 發送每日簡報
  → 提示：「近期台積電下行風險略升，建議關注」

🟠 預警模式（Orange）— 第一層 + 第三層同時觸發
  → 立即通知（電子郵件 / 手機通知）
  → 提示：「台積電高風險預警，建議評估外匯部位」

🔴 高風險模式（Red）— 第一層 + 第二層 + 第三層全部觸發
  → 最高優先通知
  → 附上詳細分析報告
  → 同步觸發 USDTWD 預警系統的「台積電風險聯動」
```

## 8. 回測框架

### 8.1 回測設計

與 USDTWD 的小樣本問題不同，台積電 4%+ 單日跌幅的歷史事件更多，統計方法可以更標準：

```
建議回測期間：2014~2024（10年，涵蓋多個景氣週期）
估計事件數量：150~200次（每年約15~20次）
統計方法：標準顯著性檢定（樣本充足）
```

### 8.2 重點驗證的歷史事件

| 事件 | 日期 | 跌幅 | 驅動類型 |
|------|------|------|---------|
| COVID 急跌 | 2020-03 | -20%（波段）| D類：美股拖累 |
| 美國晶片禁令 | 2022-10 | 單週 -15% | C類：地緣政治 |
| 科技股熊市 | 2022 全年 | -46%（波段）| A類：Fed升息+景氣 |
| AI 泡沫疑慮 | 2024-07 | 單週 -15% | B類：客戶評估重置 |

### 8.3 評估指標

| 指標 | 說明 |
|------|------|
| **Layer 1 單獨Recall** | TSM ADR 信號單獨捕捉多少事件 |
| **Layer 1+3 組合Precision** | 隔夜 + 情緒信號同時觸發的精確率 |
| **信號到事件的平均提前天數** | 評估預警時間是否足夠 |
| **不同驅動類型下的信號表現** | A/B/C/D 四類各自的命中率 |
| **C類（地緣政治）的誤報貢獻** | 評估是否需要特殊處理 |

## 9. 誠實的模型限制

延續 USDTWD 研究的嚴謹標準，以下限制必須在系統文件中明確揭露：

| 限制 | 說明 |
|------|------|
| **類型C事件無法預警** | 地緣政治衝擊無法量化預測，是系統的結構性盲點 |
| **AI 週期改變了舊有關係** | 2023年後，NVIDIA 與台積電的關聯超越傳統景氣指標，若 AI 週期結束，相關性可能崩潰 |
| **隔夜信號有效，但日內信號缺失** | 台股開盤後的盤中波動（如盤中忽然跌破 -4%）無法提前預警 |
| **選擇權數據品質** | 台灣期交所的選擇權流動性不如美股，Put/Call 比率可能受市場深度影響 |
| **法說會是週期性風險集中點** | 每季法說會前後，歷史上大部分重大跌幅集中於此，但指引是否下調本身無法預測 |
