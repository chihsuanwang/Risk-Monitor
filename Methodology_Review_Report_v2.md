## 一、最核心的問題：「60%」這個數字本身還沒被充分解讀

在繼續之前，必須先問清楚幾件事。這些不是細節問題，是影響你整套方法論結論的根本性問題。

### 問題 1A：60% 是 Recall（召回率）還是 Precision（精確率）？

這兩件事完全不同，後果也完全不同。

```
情境A：60% = Recall（召回率）
  → 20年內發生的大升值事件中，你的系統抓到了60%
  → 你漏掉了40%的大事件
  → 問題：漏報率太高

情境B：60% = Precision（精確率）
  → 你的系統每發出10次警示，有6次是對的，4次是假警報
  → 問題：誤報率太高

情境C：你說的是「Hit Rate」，但沒有定義清楚
  → 無法判斷
```

**這個問題必須明確回答，否則所有改善方向都是錯的。**

一套合格的監控系統應該呈現完整的混淆矩陣（Confusion Matrix）：

```
                  實際有大升值    實際無大升值
系統發出警示  →      TP (命中)      FP (誤報)
系統無警示    →      FN (漏報)      TN (正確沉默)

Recall    = TP / (TP + FN) = 你抓到多少比例的真實事件
Precision = TP / (TP + FP) = 你的警示有多少是真的
F1 Score  = 2 × Precision × Recall / (Precision + Recall)
```

## 二、信號結構問題：AND 邏輯是否是最優選擇？

### 問題 2：三指標 AND 邏輯的數學含義

AND 邏輯的本質是**最大化精確率（Precision），犧牲召回率（Recall）**。

```
若三個指標各自的獨立命中率：
  指標1（1M NDF > 8%）：Recall = 75%
  指標2（1M-12M > 5%）：Recall = 70%
  指標3（CVIX > MA+2σ）：Recall = 65%

若三個指標完全獨立：
  AND 邏輯的理論 Recall = 75% × 70% × 65% ≈ 34%

若三個指標高度相關（相關係數 0.8）：
  AND 邏輯的實際 Recall 接近最弱指標 = 65%
```

你觀察到的 60% 命中率，可能就是這個機制的結果。

**關鍵診斷**：計算三個指標兩兩之間的條件相關係數。

```python
# 偽代碼示意
corr_12 = df['signal_1'].corr(df['signal_2'])
corr_13 = df['signal_1'].corr(df['signal_3'])
corr_23 = df['signal_2'].corr(df['signal_3'])

# 若相關係數 > 0.6，AND邏輯的信息增益有限
# 若相關係數 < 0.3，AND邏輯有效獨立確認
```

若三個指標相關係數高，你的 AND 邏輯等同於「一個指標非常極端才觸發」，並非真正的三重確認。

### 問題 3：AND 邏輯的替代方案未被探索

你直接跳到 AND 邏輯，但更合適的框架應該是用 **Scoring 或 Voting** 的方式：

```
加權評分邏輯（示意）：
  指標1觸發 → +35分
  指標2觸發 → +35分
  指標3觸發 → +30分

  ≥ 70分 → 高風險警示（等同AND邏輯）
  50~70分 → 中風險觀察（AND邏輯完全忽略的區域）
  < 50分 → 正常
```

**AND 邏輯等於只有「高風險」和「正常」兩種狀態。**
**Scoring 邏輯可以提供分級預警，讓風控人員做差異化應對。**

如果你的目標是「提前 1 個月預警」，那麼某些中等強度的信號（只有一兩個指標觸發）可能比等到三個全部觸發更有價值。

## 三、40% 漏報率的根源診斷

這是目前最值得深挖的問題。你抓不到的那 40% 大升值事件，是隨機分布的，還是有系統性的規律？

### 問題 4：漏報事件的分類分析

**必做的診斷**：把所有漏報事件（大升值但系統沒警示）列出來，分析其特徵。

```
可能的漏報類型：

類型 A：突發性升值（黑天鵝型）
  → 升值在 3~5 天內急速發生，NDF 市場來不及反映
  → 這類漏報是正常的，無法改善，應在系統說明中標注

類型 B：CBC 主動放手升值
  → CBC 突然停止干預，導致累積壓力在短期釋放
  → NDF 信號可能已觸發，但 CVIX 沒有配合（台幣升值是正面因素，Vol未必上升）
  → 改善方向：加入「CBC干預強度突然下降」的偵測邏輯

類型 C：基本面驅動的緩慢升值
  → 台灣出口長期強勁，出口商積累結匯壓力
  → NDF 成本可能只是溫和上升，不到觸發閾值
  → 改善方向：加入外銷訂單趨勢指標

類型 D：特定時間段集中漏報
  → 若 40% 漏報集中在某幾年，代表那些年份的市場結構讓你的模型失效
  → 改善方向：加入市場結構制度因子
```

**沒有這個分析，任何改善方向都是猜測。**

### 問題 5：CVIX 指標可能產生系統性偏誤

CVIX（DB FX Volatility Indicator）本質上衡量的是**全球外匯市場波動率**，並非 USDTWD 的特定波動。

**潛在問題**：台幣升值是一件相對「正面」的風險事件（對外資而言），但 CVIX 通常在**負面衝擊**（Risk-off）時才飆高。

```
思考情境：
  全球股市大漲 + 外資大買台股 → 台幣升值壓力巨大
  但此時 CVIX 可能很低（市場平靜）
  → NDF 1M 高、NDF 利差大，但 CVIX 沒觸發
  → 系統發出中等信號，但因 AND 邏輯，最終無警示
  → 這是結構性的漏報來源
```

**建議檢驗**：統計所有漏報事件中，CVIX 是否是最常沒有觸發的那個指標。如果是，這個指標在台幣升值的情境下可能是個障礙，而不是幫手。

## 四、統計方法論的剩餘問題

### 問題 6：多重比較問題（Multiple Testing Problem）

你研究了 5 大類指標，每類可能有多個具體指標，最終選出 3 個。

這個「選指標」的過程本身就有統計風險：

```
若你測試了 50 個指標：
  在 p < 0.05 的顯著水準下，純靠運氣也會有 50 × 5% = 2.5 個指標看起來「顯著」

若你測試了 20 個指標：
  也會有 1 個指標是假陽性

你選出的 3 個指標，至少應通過 Bonferroni 校正：
  調整後顯著水準 = 0.05 / 測試指標數量
```

**你是否記錄過你總共測試了多少個指標？**
若測試了 30 個以上，三個指標的顯著性需要重新評估。

### 問題 7：信號觸發的時序與衰減未被研究

AND 邏輯假設三個指標「同時」觸發。但在實際數據中：

- 三個指標是否通常在同一天觸發？
- 還是有「第一個先觸發，等 2~3 週後另外兩個才跟上」的序列？

**若存在序列性觸發，AND 邏輯會人為縮短你的預警時間。**

```
實際案例（假設）：
  T-20 : NDF 利差觸發
  T-10 : 1M NDF 成本觸發
  T-5  : CVIX 觸發  ← AND 邏輯在此刻才發出警示
  T+0  : 大升值開始

  你以為有 1 個月預警，實際上只剩 5 天
```

**建議**：繪製三個指標各自觸發的時間點，相對於大升值事件的分布直方圖，找出哪個指標最早、哪個最晚。

### 問題 8：閾值的時間穩定性未被驗證

你的閾值（5%、8%、MA+2σ）是用 15 年訓練期優化出來的。

但 NDF 市場本身的結構在這 20 年內發生了顯著變化：
- **2003~2008**：NDF 市場流動性有限，成本基準線較高
- **2015~2020**：亞洲 NDF 市場深化，成本中樞下移
- **2020~今**：Fed 升息週期讓絕對利率水準大幅改變

**問題**：在 Fed 基準利率 5% 的環境下，「1M NDF 年化成本 > 8%」的意涵，和 Fed 利率 0% 的環境是完全不同的。

8% 在低利率時代是「非常高」，但在高利率時代可能只是「略高於基準」。

**建議**：
- 計算 1M NDF 年化成本相對於**同期美元無風險利率**的超額溢價
- 閾值應該是「超額溢價 > X%」，而非絕對值 > 8%
- 或者用動態閾值：像 CVIX 一樣，用移動平均 + 標準差取代固定數字

## 五、被丟棄的指標值得再審

### 問題 9：競爭對手匯率為何被排除？

你研究了日圓、韓元、人民幣、歐元，但最終沒有納入。

這個決定需要更多解釋，因為亞幣連動是一個真實存在的現象：

```
日圓 → 台幣的領先關係（案例）：
  2022年：日圓率先大幅貶值，韓元跟隨，台幣反應最慢
  2024年：日圓升值預期 → 亞幣普遍升值 → 台幣跟漲

USDKRW 是亞洲外匯市場中流動性最高的匯率之一，
往往比 USDTWD 先反映全球資金流動方向。
```

**問題**：競爭對手匯率的排除，是因為預測力真的不足，還是因為：
1. 數據不夠完整（有缺值）
2. 單獨看不夠，但作為確認指標可能有用
3. 和 NDF 指標重疊（同樣都在衡量亞幣壓力）

建議重新評估 **USDKRW 的2週領先相關性**，以及 **USDJPY 在 Risk-off 情境下的配合度**。

### 問題 10：資金面指標（外資買超）的角色被低估

外資在台股的淨買超，是 USDTWD 最直接的資金流動信號。你研究了它，但沒有納入最終指標。

**可能的原因與再評估**：

如果外資買超「單獨的預測力不足」，這是可以接受的。但：
- 外資買超在「NDF 信號已觸發」的條件下，是否有**條件性的增強效果**？
- 即：在 NDF 信號高位時，若外資同步大量買超，Hit Rate 能否從 60% 升至 75%？

這是一個**條件概率問題**，不是獨立預測力問題。建議做條件性分析：

```
P(大升值 | AND信號觸發 AND 外資買超>500億) = ?
P(大升值 | AND信號觸發 AND 外資淨賣超) = ?

若兩者差距顯著，外資買超是一個有效的「信號強度調整因子」
```

### 問題 12：信號冷卻機制不明

一旦系統發出警示後，它何時「重置」？

```
潛在問題：
  T日 → 三指標觸發，發出警示
  T+1日 → 指標仍在高位，再次觸發？

```

這個機制的設計直接影響 60% Hit Rate 的計算方式，如果沒有適當的冷卻期設定，Hit Rate 的定義本身就有問題。
